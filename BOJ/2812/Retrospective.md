## 2) 나의 풀이
- 날짜: 260324
- 풀이 횟수: 1
### 아이디어
- 접근 전략
```text
완탐으로 최소값 찾으면서 하나씩 제거하는 방법이 있으나 O(N^2)임.
O(N)으로 해결할 수 있는게 모노스틱 스택.
s.top()을 현재 인덱스 수와 비교해 s.top()이 작으면 제거
-> 이런식으로 최소값들 다 제거하고 큰 값만 남김.
스택이라 순서 반대로 저장되므로 한번 뒤집어서 출력
```
- 왜 이 방법을 먼저 떠올렸는지
```text
- 완탐이 떠오르는데 O(N^2)이다? -> 모노토닉 스택 후보군으로 떠오름.
- 펄스에서 모노토닉 스택 배운지 얼마 안돼서 잘 떠오름.
```

### 구현 포인트
- 사용한 자료구조/알고리즘: Monotonic Stack
- 시간복잡도: O(N)
- 코드에서 실수하기 쉬운 부분: 모노토닉 스택의 템플릿 구현
- 까다로웠던 제약 또는 edge case: 단조감소 or 같은 수만 나오는 수열(541, 888 ...)

### 결과
- 제출 횟수: 2
- 제출 결과(AC/WA/TLE/MLE/RE): WA - AC
- 틀린 원인: 위의 edge case에 걸림.
- 수정 내용: 다음의 코드 추가
```C++
	while(cnt < k){
        s.pop();
        cnt++;
    }
```
단조감소 수열 등이 s.top() < i의 조건에 걸리지 않아서 k만큼 수를 못 뺌.

대신 cnt가 계속 추적중이므로 while문 종료 후 다시 cnt와 k가 맞는지 검사하는 while문 추가
